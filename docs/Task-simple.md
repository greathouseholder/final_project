Какие задачи может выполнять приложение:


1) создать бота, в котором компании добавляют свои базы знаний, которые можно будет использовать как для QA 
2) бот отделен от fastapi приложения 
3) компания создает свою коллекцию в базе данных по api запросу 
4) дальше можно загрузить данные в векторную базу в коллекцию компании, предоставив в json формате по заранее известной схеме (text, metadata)
5) при загрузке задействуется эмбеддер
6) когда данные загружены можно обращаться через бота к сервису и спрашивать вопросы, они прогоняются через эмбеддер, после чего выдаются 5 ближайших по расстоянию документов, после чего через LLM проводим суммаризацию
7) в боте пользователь может быть либо админом, либо юзером. Юзер задает вопросы, админ добавляет данные в базу знаний.
