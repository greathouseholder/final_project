# Устройство проекта

## tests/

Тесты пишутся в специальной папке через фреймворк pytest. 

Основная цель теста - покрыть угловые кейсы работы компонента, а также проверить основную задачу.

Тесты бывают трех видов: юнит (отдельные функции), модульные и интеграционные (связь компонентов). 
Мы будем писать в основном модульные и интеграционные, так как на отдельные функции потратим кучу времени.

## src/

Наше приложение поделено на слои. Удобнее будет представить связи через диаграмму:

https://boards.yandex.ru/whiteboard/?hash=90d6b52cc77eadcecc24de0db025c4fd
